# Glossary

### Glossary

**API (Application Programming Interface)** – A set of protocols and tools for building software and allowing different applications to communicate with each other.

**Asset** – Any hardware, software, or network component tracked within IT Asset Management (ITAM) systems.

**ArcGIS Enterprise** – A comprehensive GIS platform that allows organizations to manage, analyze, and share geographic data on-premises or in a hybrid environment.

**ArcGIS Indoors** – Esri’s indoor mapping solution for managing facilities and spatial data, integrated with ITAM and operational workflows.

**ArcGIS Online (AGOL)** – A cloud-based GIS platform from Esri that provides mapping, analysis, and collaboration capabilities.

**Backup** – The process of creating copies of data or systems to protect against data loss or corruption.

**BI (Business Intelligence)** – Techniques and tools for analyzing data to support decision-making and business planning.

**Change Management** – The process of controlling and documenting changes to IT systems, typically managed through ITSM platforms.

**Cloud Computing** – The delivery of computing services over the internet, including storage, processing, and software.

**Configuration Item (CI)** – Any component within ITSM that needs to be managed in order to deliver services.

**Data Dictionary** – A centralized repository of metadata, defining data elements, formats, and relationships.

**Dashboard** – A visual interface displaying real-time data, metrics, and KPIs for monitoring performance or system status.

**Deployment** – The process of installing, configuring, and enabling a system or application for use.

**DevOps** – A set of practices that combines software development and IT operations to shorten delivery cycles and improve reliability.

**Documentation** – Written records that describe system architecture, usage, and processes for reference or training.

**Enterprise Architecture** – A framework for aligning IT infrastructure and operations with organizational goals and strategy.

**Field List** – In RST or MyST documentation, a structured set of key-value pairs used to describe metadata, properties, or attributes.

**GIS (Geographic Information System)** – A system designed to capture, store, analyze, and manage spatial or geographic data.

**Integration** – The process of connecting multiple systems or platforms to work together seamlessly.

**Incident** – An unplanned interruption or reduction in the quality of an IT service.

**IP Address** – A numerical label assigned to devices on a network for identification and communication.

**IT Asset Management (ITAM)** – The practice of tracking and managing IT assets throughout their lifecycle.

**IT Service Management (ITSM)** – A set of processes for designing, delivering, managing, and improving IT services.

**Jupyter Notebook** – An interactive web-based environment for running Python code, visualizing data, and documenting workflows.

**License** – A legal agreement specifying permissions and restrictions for using software or other digital assets.

**Maintenance** – Ongoing activities to keep systems operational, secure, and up to date.

**Metadata** – Data that describes other data, providing context, structure, or additional information.

**Network Topology** – The arrangement and interconnection of devices within a network.

**Patch** – A software update designed to fix vulnerabilities, bugs, or performance issues.

**Platform** – A technology stack or environment on which applications, services, or integrations run.

**Problem Statement** – A concise description of the issues or challenges that a framework, solution, or project addresses.

**Project Plan** – A document outlining goals, tasks, timelines, and responsibilities for a project.

**Python** – A high-level programming language commonly used for scripting, automation, and data analysis.

**Reference Materials** – Supplemental guides, checklists, or cheat sheets that provide quick access to essential information.

**Reporting** – The process of generating structured output from data to summarize status, performance, or trends.

**REST API** – A type of API that uses HTTP requests to GET, POST, PUT, or DELETE data between systems.

**ServiceNow** – A cloud-based ITSM platform that manages incidents, service requests, changes, and asset workflows.

**Script** – A program or set of instructions that automates tasks within an environment.

**Security Group** – A set of permissions that defines access to systems, applications, or data.

**SLA (Service Level Agreement)** – A formal contract specifying the expected quality, availability, and performance of a service.

**Spatial Data** – Information about the location, shape, and relationships of physical objects or features.

**Synchronization** – The process of keeping data consistent and up to date across multiple systems.

**Table** – A structured arrangement of data in rows and columns, used for documentation or analysis.

**Template** – A preformatted document or file used as a starting point for consistent project deliverables.

**Troubleshooting** – The process of identifying, diagnosing, and resolving issues or failures.

**Validation** – The process of ensuring that data, configurations, or processes meet defined requirements.

**Versioning** – The practice of tracking and managing changes to software, documents, or datasets over time.

**Workflow** – A sequence of tasks or steps that define how a process is executed or managed.

**XML (Extensible Markup Language)** – A markup language used to encode data in a structured, machine-readable format.

**YAML (YAML Ain’t Markup Language)** – A human-readable data serialization format commonly used for configuration files and automation scripts.
